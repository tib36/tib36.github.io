VSC+PHPdebug填坑日记

之前因为重装系统，原本搭建的php调试环境没了
本来还没什么问题的，但是最近在研究ThinkPHP，需要手动跟流程，那就必须要搞个调试环境
在新的虚拟机里配置这东西发现异常痛苦，踩到各种诡异的坑
而且由于是新环境，根本不知道问题出在哪里
最要命的是根本触发不到断点，改了半天最后放弃，去拷贝别人的配置文件，改改凑合用

这里备份一下配置文件，以备后用

launch.json

{
    // 使用 IntelliSense 了解相关属性。 
    // 悬停以查看现有属性的描述。
    // 欲了解更多信息，请访问: https://go.microsoft.com/fwlink/?linkid=830387
    "version": "0.2.0",
    "configurations": [
        {
            "name": "Listen for XDebug",
            "type": "php",
            "request": "launch",
            "port": 9179,
            "stopOnEntry":true,
            "log": true
        },
        {
            "name": "Launch currently open script",
            "type": "php",
            "request": "launch",
            "program": "${file}",
            "cwd": "${fileDirname}",
            "port": 9179,
            "stopOnEntry": false,
            "log":true
        }
    ]
}

php.ini（尾部）

[Xdebug]
zend_extension=C:/phpstudy_pro/Extensions/php/php7.3.4nts/ext/php_xdebug.dll
xdebug.trace_output_dir=C:/phpstudy_pro/Extensions/php_log/php7.3.4nts.xdebug.trace
xdebug.profiler_output_dir=C:/phpstudy_pro/Extensions/php_log/php7.3.4nts.xdebug.profiler
xdebug.profiler_append = 0
xdebug.profiler_enable = 1
xdebug.profiler_enable_trigger = 0
xdebug.profiler_output_name = "cache.out.%t-%s"
xdebug.remote_enable = 1
xdebug.remote_handler = "dbgp"
xdebug.remote_host = "127.0.0.1"
xdebug.remote_autostart = 1
xdebug.remote_autorestart=1
xdebug.remote_connect_back=1
xdebug.remote_port=9179
zend_extension=php_xdebug.dll

然后在Xdebug的启动选项中选择Launch currently open script
启动即可
用起来很怪，但是勉强能用
